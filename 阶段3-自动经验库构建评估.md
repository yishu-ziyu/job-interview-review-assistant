# 阶段3：自动经验库构建评估（产品 + 工程）

更新时间：2026-02-15

## 1. 目标重述

你希望从“手动录入为主”升级为“自动输入为主 + 手动补充为辅”。  
产品目标是构建可持续增长、可追溯、可用于面试前策略生成的经验库。

## 2. 结论先行

这个方向可做，但必须走三层架构：

1. 授权数据源层：官方 API、自有数据、用户授权上传
2. 研究情报层：Deep Research API 结果导入（带引用）
3. 风险隔离层：非授权网页内容只做人审输入，不做主干自动抓取

推荐路线：先做 1 + 2，让自动输入占比先到 40%-60%，再考虑更重自动化。

## 3. 你关心的关键问题：能不能“直接调取别家 Deep Research 结果”？

可以分成两种情况：

1. 通过官方 API 发起 Deep Research 任务：可行  
2. 直接读取消费端产品页面结果（例如在网页里抓 Gemini/GPT/豆包结果页）：不建议作为主路线

原因：

1. 工程稳定性差：页面结构常变
2. 合规边界不清：条款和授权通常不支持“页面抓取后再分发”
3. 可追溯性差：难做标准化结构和长期运维

结论：使用“官方 API 调研 + 结构化导入”替代“抓结果页”。

## 4. 自动输入的完整链路（不是只抓文本）

1. 任务编排：按岗位/公司生成研究任务
2. 数据采集：API 调研、用户上传、复盘导入
3. 结构抽取：统一成 question/pitfall/betterAnswer 结构
4. 质量评分：来源可信度、具体性、时效性
5. 去重合并：相似问题聚类，保留最佳答案版本
6. 入库索引：关键词 + 标签 + 向量（后续）
7. 召回生成：面试前策略摘要/高概率问题/风险点/行动清单
8. 反馈回写：用户是否采纳、结果如何

你当前系统已经覆盖了 6-7 的基础版本。

## 5. 数据源评估矩阵（自动化优先级）

### A. 牛客（优先）

- 价值：岗位和面试经验密度高
- 自动化可行性：中到高（取决于是否拿到可用 API 权限）
- 证据：官方文档提供企业 API、OAuth2、ATS 对接能力
- 风险：个人开发阶段可能拿不到企业级权限

### B. 小红书（先人机协同）

- 价值：真实经验丰富，细节强
- 自动化可行性：中低
- 证据：可见的官方开放能力更偏营销生态（广告/营销 API）
- 风险：内容多为图文混合，结构化提取成本高
- 建议：优先做“用户上传截图 + OCR + 人审确认”而不是直爬

说明：上面对“小红书无公开面经内容 API”的判断是基于当前可公开检索到的官方资料做出的推断，不是平台官方明示结论。

### C. 海外站点（例如 Glassdoor）

- 价值：公司维度问题覆盖广
- 自动化可行性：低（未经授权）
- 证据：Terms 明确限制 scrape/strip/mine；robots 对多类 AI crawler 禁止
- 建议：不作为自动抓取主源

### D. Deep Research API（推荐做中间层）

- 价值：能拿到“带引用”的研究结果，适合入库前处理
- 自动化可行性：中高
- 证据：OpenAI 提供 deep research API；Gemini deep research API 有 allowlist 机制
- 风险：成本、时延、配额和权限差异
- 建议：把它定位为“情报采集器”，不是直接真值库

## 6. 类似工作：成功经验与失败反思

### 成功样本（可借鉴）

1. DuerQuiz（KDD 2019）  
   论文写明系统已在 2018 校招场景部署，核心是“结构化技能图谱 + 个性化问题推荐”。  
   可借鉴：先把知识结构化，再做推荐和生成。

2. 智谱“AI 模拟面试官”实践  
   官方最佳实践是“先建知识库，再做检索生成问答”。  
   可借鉴：你的产品路径和这条成熟路径一致。

### 常见失败模式（基于公开实践的归纳）

1. 只追求抓取规模，不做质量闭环  
   结果：库越大越乱，策略不可信。

2. 忽略来源授权与平台规则  
   结果：接口封禁、链路中断，项目不可持续。

3. 不做证据链和可追溯  
   结果：用户不敢采用系统建议，留存低。

## 7. 阶段化落地（按你每周 10-20 小时节奏）

### Phase 3A（1-2 周）：自动导入 MVP（推荐立刻做）

- 目标：自动输入占比 30%-40%
- 实现：
  - 增加“研究结果导入”入口（支持粘贴带引用文本 / JSON）
  - 做结构化抽取器（question/pitfall/betterAnswer/tags/sourceUrl）
  - 做基础去重（问题文本 + 公司 + 岗位）

### Phase 3B（2-4 周）：自动采集增强

- 目标：自动输入占比 50%-60%
- 实现：
  - 接入可用授权源 API（优先牛客）
  - 定时增量同步
  - 增加人审抽检台（抽样确认准确率）

### Phase 3C（后续）：多模态与高质量检索

- 目标：策略质量稳定提升
- 实现：
  - 图片 OCR 入库
  - 向量召回 + 重排
  - 证据追溯（每条经验都可回到来源）

## 8. 难度评估

1. 手动录入/复盘导入：低
2. Deep Research 结果导入：中
3. 授权 API 接入：中
4. 图片面经 OCR + 清洗：中高
5. 未授权全网直爬：高且高风险（不建议）

## 9. 阶段 3 验收指标（PM 视角）

1. 自动入库占比（Auto Ingestion Ratio）
2. 去重后有效条目占比（Useful Entry Ratio）
3. 策略被采纳率（Action Adoption）
4. 采纳后面试“卡顿率”下降（Outcome Improvement）
5. 建议可追溯率（是否可回溯到来源证据）

## 10. 选择题决策（你直接选）

1. 路线 A（推荐）：先做 Deep Research 结果导入 + 去重评分 + 人审抽检  
2. 路线 B：先冲授权 API（牛客），导入器后置  
3. 路线 C：先做图片 OCR 入库（适合你已经有大量截图素材）
