# 求职面试复盘助手：完整使用指南与产品完成度评估（v1）

更新时间：2026-02-28
适用版本：当前本地 Web 版本（Next.js 16）

## 1. 产品目标与适用人群

产品目标：
- 帮用户把“面试后记忆”快速转成结构化复盘。
- 把分散的面经/研究结论沉淀为可检索经验库。
- 在面试前，基于经验库与多源研究输出准备策略，并通过质量门槛控制风险。

适用人群：
- 求职中的个人用户（重点：产品经理岗位，也支持其他岗位）。
- 需要做“复盘 + 面试前准备”的轻量工作流用户。

当前交付形态：
- Web 端（本地运行）。

## 3. 用户完整使用路径（建议顺序）

### 路径 A：面试后复盘 -> 入库

1. 进入“复盘模式”。
2. 输入岗位/公司/轮次/面试回忆文本。
3. 点击生成三段式复盘（摘要/问题清单/下次行动）。
4. 点击“将本次复盘写入经验库”。
5. 在“经验库条目”确认新增内容。

价值：
- 形成个人高质量样本，为后续“面试前策略”提供真实依据。

### 路径 B：面试前准备（经验库驱动）

1. 进入“面试前策略生成”。
2. 输入目标岗位、目标公司、关注点。
3. 保持“质量门槛开关”开启（建议阈值 60）。
4. 点击生成策略。

系统输出：
- 策略摘要
- 高概率问题与应答
- 风险提醒
- 行动清单
- 策略质量体检（证据得分/等级/冲突分布/门槛通过状态）

门槛未通过时：
- 系统会优先给“补证据动作”，而不是直接放行策略。

### 路径 C：外部研究结果导入经验库

#### C1 手动导入

1. 粘贴研究报告文本。
2. 粘贴来源 URL（每行一个）。
3. 点击导入。
4. 查看来源复查状态（supported/weak/conflict/unreachable）。

#### C2 B1 书签抓取导入

1. 在页面中复制书签脚本 URL。
2. 浏览器新建书签并替换地址。
3. 打开豆包/Minimax/ChatGPT/Gemini 的研究结果页并点击书签。
4. 回到系统点击读取抓取结果。
5. 检查自动补全建议（岗位/公司）并导入。

#### C3 B1 全自动抓取导入（实验）

1. 输入研究页 URL。
2. 点击“一键自动抓取并导入经验库”。
3. 查看抓取元信息、来源复查、入库统计。

### 路径 D：B2 多源 Deep Research 画像

1. 输入目标岗位/公司/关注点。
2. 选择每渠道来源数。
3. 开启 B2.1 反思式二次检索（建议开）。
4. 按需开启 B2.3 多模型交叉验证。
5. 建议开启 B2.6 异步任务队列（长任务更稳）。

新增能力：
- B2.4 多检索引擎回退：主检失败自动切备用引擎。
- B2.5 论点级证据聚类：同一结论至少 2 域名支持才视为有效论点。
- Research Quality Gate：结果可用性门禁（通过/未通过 + 阻塞项 + 修复建议）。

## 4. 核心能力说明（工程视角）

### 4.1 质量门槛（面试前策略）

输入：
- `qualityGateEnabled`
- `qualityGateThreshold`（40-90）

判定逻辑：
- 证据分低于阈值 -> 不通过。
- 存在冲突条目 -> 不通过。

结果：
- 不通过时阻断“正式策略放行”，先引导补证据。

### 4.2 研究门禁（Deep Research）

判定信号：
- 来源平均分
- A级来源数量
- 覆盖渠道数
- 独立域名数
- 多模型一致度（启用时）
- 合格论点数（双域名门槛）

结果：
- `readiness.gatePassed` 决定是否可直接依赖该画像。

### 4.3 异步队列（B2.6）

状态机：
- `queued -> running -> completed`
- 失败进入 `retrying`，耗尽后 `failed`
- 用户可取消到 `cancelled`

能力：
- 自动重试（指数退避）
- 可追踪事件（events）
- 适合长任务和偶发网络波动

## 7. 常见问题与排查

问题：Deep Research 很慢
- 开启 B2.6 异步队列。
- 降低每渠道来源数（如 6）。
- 关闭非必要的交叉验证。

问题：抓取到聊天页但没有有效外链
- 优先抓“含参考资料列表”的页面状态。
- 使用手动 URL 补充来源。

问题：策略经常被门槛拦截
- 先补充 supported 条目数量。
- 优先处理冲突条目。
- 提升来源多样性（self/community/other）。
