# 技术实现说明 v0.1

更新时间：2026-02-15

## 1. 架构概览

- 前端页面：输入面试回忆并展示三段式输出
- API 路由：`POST /api/reviews`
- LLM 适配层：根据 `LLM_PROVIDER` 调用智谱 / Minimax / 豆包
- 数据持久化：可选写入 Supabase `interview_reviews`

## 2. 关键文件

- 页面：`app/src/app/page.tsx`
- API：`app/src/app/api/reviews/route.ts`
- 复盘逻辑：`app/src/lib/review.ts`
- LLM 调用：`app/src/lib/llm.ts`
- Supabase 记录：`app/src/lib/supabase-log.ts`
- 数据类型：`app/src/lib/types.ts`

## 3. 请求流程

1. 用户在页面提交 `targetRole/company/round/rawNotes`
2. API 校验输入长度与必要字段
3. 调用 LLM 生成严格 JSON
4. 服务端校验并标准化输出
5. 返回三段式结果到前端
6. 若配置 Supabase，则异步写入数据库

## 4. 稳定性设计

- 强制 JSON 输出并做二次解析
- 字段级校验，防止模型输出异常格式
- Supabase 写入失败不阻断主流程
- 默认脚本使用 webpack，规避 Turbopack 环境差异

## 5. 下一步建议

- 接入语音转文本作为输入前置层
- 加历史记录列表与筛选
- 设计“同岗位高频题”生成能力（基于积累数据）
