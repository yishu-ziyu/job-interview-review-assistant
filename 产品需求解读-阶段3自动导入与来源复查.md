# 产品需求解读（阶段 3A：自动导入 + 来源复查）

更新时间：2026-02-15

## 1. 这次解决了什么问题

上一阶段的问题：经验库主要靠手动录入，自动化弱。  
这一阶段的目标：
1. 把 Deep Research 结果批量转成经验条目，并给每条条目打“证据状态”。
2. 做 A2 去重合并，减少重复噪声。
3. 做 A3 图片 OCR 导入，处理截图类面经素材。
4. 做 B1 浏览器一键抓取，降低“人工搬运”成本。
5. 做 B2 多源并行 Deep Research，自动构建岗位画像。
6. 做 A0 内置 PM Demo 经验库，一键演示“面试前策略”全流程。

一句话定义：
“把研究报告从‘阅读材料’变成‘可检索、可复用、可追溯’的面试资产。”

## 2. 用户怎么用（操作流）

### 路径 A2：经验库去重合并

1. 进入 `面试前准备模式`
2. 在 `A2｜经验库去重合并` 点击 `预览去重结果`
3. 确认后点击 `执行去重合并`

系统输出：
- 去重前后条目数量
- 合并条目数
- 重复组样本（用于你做产品判断）

### 路径 A0：一键加载内置 PM Demo 库（100条/10公司）

1. 进入 `面试前准备模式`
2. 点击 `一键加载 PM Demo 库（推荐）`
3. 系统自动导入内置条目并去重

系统输出：
- 本次新增条目数
- 跳过条目数（避免重复导入）
- 当前经验库总条数与内置条目数

### 路径 A3：图片 OCR 导入

1. 进入 `面试前准备模式`
2. 在 `A3｜图片 OCR 导入` 上传截图
3. 输入岗位、公司（可选）、来源 URL（可选）
4. 点击 `上传图片并导入经验库`

系统输出：
- OCR 文本长度
- 入库条目数
- 证据状态统计
- OCR 文本预览（前 800 字）

### 路径 A1：导入研究结果

1. 进入 `面试前准备模式`
2. 找到 `Deep Research 结果导入`
3. 选择研究来源（Gemini/GPT/豆包/智谱/其他）
4. 输入目标岗位、公司、轮次（可选）
5. 粘贴来源 URL（每行一个）
6. 粘贴研究报告正文
7. 点击 `导入研究结果到经验库`

系统输出：
- 新增条目数量
- 来源复查统计（证据支持/偏弱/冲突/待复核/不可达）
- 来源复查详情（URL 是否可访问、页面标题）

### 路径 B1：浏览器一键抓取（推荐）

1. 进入 `面试前准备模式`
2. 在 `B1｜网页一键抓取 + 研究结果导入` 点击 `复制书签脚本 URL`
3. 在浏览器中新建书签并替换地址为该 URL
4. 打开豆包/Minimax/ChatGPT/Gemini 研究结果页
5. 若为豆包，先展开右侧 `参考资料`，再点击该书签（优先抓外链）
6. 回到系统点击 `读取抓取结果（剪贴板）`
7. 系统自动回填 `研究来源 / 来源 URL / 报告文本`
8. 点击 `导入研究结果到经验库`

系统输出：
- 抓取字数、来源链接数量、识别来源类型
- 文献外链状态（已提取 / 未提取）
- 自动回填后的导入结果（新增条目 + 复查统计）

### 路径 B1.1：抓取后清洗与可解释预览

在 B1 的第 5 步后，系统会新增：
1. 自动清洗：去掉常见网页噪声（导航按钮、重复短句等）
2. 清洗指标：展示 `原始字数 -> 清洗字数`、`去噪行数`
3. 文本对比：可展开查看“清洗后预览”和“原始抓取预览”
4. 人工控制：可切换 `使用清洗文本` 或 `使用原始文本` 再导入

### 路径 B1.2：自动补全岗位/公司建议

在读取抓取结果后，系统会基于 `页面标题 + 抓取文本 + 来源链接` 自动识别：
1. 目标岗位建议（如：AI 产品经理 / 数据产品经理）
2. 目标公司建议（如：字节跳动 / 腾讯 / 华为）
3. 识别置信度（高/中/低）

交互方式：
1. 点击建议标签可单项应用
2. 点击 `一键应用建议` 可同时回填岗位与公司
3. 若原表单为空或为默认值，系统会先自动尝试填一版建议

### 路径 B1.3：公司别名词典可配置面板

在 B1 卡片中新增 `B1.3 公司别名词典配置`：
1. 可新增公司条目（公司名 + 别名列表）
2. 可编辑已有条目（公司名、别名）
3. 可删除条目
4. 可保存到本地数据文件
5. 可一键恢复默认词典

业务价值：
1. 让 B1.2 的“公司自动补全”可被用户持续调优
2. 避免“写死词典”导致的识别偏差
3. 提供 PM 可解释的“识别规则入口”

### 路径 B2：多源并行 Deep Research 岗位画像

1. 输入目标岗位、目标公司（可选）、关注点（可选）
2. 点击 `生成岗位 Deep Research 画像`
3. 系统并行执行 5 条检索通道：
`job(岗位JD)` / `interview(面经)` / `community(社区)` / `knowledge(方法论)` / `salary(薪酬信号)`
4. 自动聚合来源后输出岗位画像：
- 岗位摘要
- 核心职责
- 核心技能
- 面试高频主题
- 市场信号
- 风险与盲区
- 行动清单
5. 可一键回填到 B1 导入区再入经验库

### 路径 A4：在经验库消费结果

导入后，每条条目会显示：
- 复查状态
- 复查备注
- 复查置信度
- 来源链接

然后你仍可用 `生成面试前准备策略` 把这些条目融合成策略。

## 4. PM 视角验收标准（阶段 3A）

1. 自动导入成功率 >= 80%
2. 来源可访问率 >= 60%（受外站限制，先可接受）
3. 导入后可用条目比例（非空 question/pitfall/betterAnswer）>= 90%
4. 用户在生成策略时使用“已复核条目”占比持续上升

## 5. 当前边界（你需要知道）

1. 这不是全网爬虫，只处理你提供的来源 URL
2. 页面复查基于可访问文本，部分站点会拦截
3. 复查标签是“机器判断 + 可人工复核”，不是法律意义真伪裁定
4. OCR 当前是单图导入（<=5MB），还不支持批量队列
